# [Project] Mind Balance - 성격 유형 테스트(MQ) 구현 기획서

기존 Mind Balance 사이트에 새롭게 추가될 '성격 유형 테스트' 페이지 구현을 요청합니다.
이 테스트는 저작권 이슈를 피해 독자적인 명칭을 사용하며, 유저의 선택에 따라 16가지 결과 중 하나를 노출합니다.
특히 메인 화면 및 결과 페이지에서 'Essence(본질지향)' 모드와 'Concept(컨셉지향)' 모드를 버튼 토글로 스위칭할 수 있어야 합니다.

## 1. 기술 스택 및 구조 (Architecture)
- UI 컴포넌트와 질문/결과 데이터(JSON)를 철저히 분리하여 구현해 주세요.
- 디자인 톤앤매너는 기존 사이트의 모던하고 깔끔한 스타일에 맞춥니다.

## 2. 핵심 로직 및 상태 관리 (State Management)
- **모드 상태 (`currentMode`):** `'essence'` 또는 `'concept'` (디폴트: 'essence')
  - 테스트 메인 화면의 제목 밑에 버튼/토글 UI를 배치하여 유저가 모드를 전환할 수 있게 합니다.
- **점수 카운팅 (`scores`):**
  - 유저가 답변을 고를 때마다 `E, I, S, N, T, F, J, P` 각각의 변수에 스코어를 누적합니다.
  - 질문은 총 20문항이며, 카테고리별로 5문항씩 고정 배정됩니다.
- **결과 산출:**
  - `E >= I` 이면 첫 글자 'E' (미만이면 'I')
  - `S >= N` 이면 두 번째 글자 'S' (미만이면 'N')
  - `T >= F` 이면 세 번째 글자 'T' (미만이면 'F')
  - `J >= P` 이면 네 번째 글자 'J' (미만이면 'P')
  - 예: 최종 스코어가 조합된 `ESTJ` 코드를 도출하여 결과 데이터와 매칭합니다.

## 4. 요구사항 (Requirements)
1. 질문 단계에서는 상단의 `currentMode`에 따라 `question_essence` 또는 `question_concept` 텍스트가 화면에 렌더링되어야 합니다.
2. 결과 페이지에서도 동일하게 상단 토글 버튼을 통해 'Essence 결과 뷰'와 'Concept 결과 뷰'를 실시간으로 스위칭하여 볼 수 있어야 합니다. (유저가 푼 결과 코드는 유지됨)
3. 결과 페이지 하단에 [카카오톡 공유하기], [링크 복사] 기능을 구현해 주세요.
